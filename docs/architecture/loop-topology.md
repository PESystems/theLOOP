# LOOP Topology

![LOOP topology](../../assets/loop-topology.svg)

## Reading the diagram

At a high level, LOOP separates:
- **The Node** for shared coordination truth
- **The Container** for doctrine and behavior-shaping context
- **The Floor** for real execution and side effects

It also separates roles:
- **Theo** for strategy, critique, and architecture pressure-testing
- **Lenny’s Brain** for planning, routing, synthesis, and controlled low-risk action
- **Cowork / Claude Code** for structured execution on the Floor

## Why this matters

The model is trying to protect a few boundaries:

- planning is not execution
- shared state is not the same thing as behavioral doctrine
- execution artifacts should not silently become canonical truth
- the same thing can exist in more than one layer only when each copy is doing a different job
