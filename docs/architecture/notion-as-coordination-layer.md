# Notion as Coordination Layer

One of the more important choices in LOOP is using Notion as **The Node** rather than as the whole system.

## The claim

Notion is best used here as:
- shared coordination truth
- inspectable project and task state
- dashboards and handoffs
- decision logs
- concise summaries and pointers

It is **not** asked to be:
- the reasoning layer
- the execution layer
- the file system
- the warehouse of every draft, transcript, and artifact

That separation is part of what makes LOOP different.

## Why this matters

A lot of AI workspace designs quietly collapse too many jobs into one tool.
That usually feels efficient at first and messy later.

When Notion is forced to become:
- the brain
- the archive
- the execution log
- the draft warehouse
- the source of every rule

it becomes harder to:
- inspect current state cleanly
- keep shared records concise
- preserve execution depth elsewhere
- prevent duplication and drift

LOOP tries to avoid that by giving Notion a narrower, stronger role.

## The Node’s actual job

Inside LOOP, The Node should answer:
- what exists
- what matters now
- what changed
- what another session or operator should see quickly

That makes it a coordination layer, not a universal memory substrate.

## What belongs in the Node

Examples:
- project and task records
- client relations
- dashboards and current-state views
- concise architecture notes
- handoffs
- decision logs
- artifact pointers
- inspectable summaries

## What should stay elsewhere

### Container
Keep here:
- doctrine
- skills
- nomenclature
- behavior-shaping rules
- execution prompt specs

### Floor
Keep here:
- deep drafts
- local files
- scripts
- generated outputs
- execution packets
- receipts
- work-in-progress

That split preserves the core doctrine:
- Node = coordination truth
- Container = behavioral truth
- Floor = execution truth

## Public significance

This is worth calling out because a lot of people are talking about agents, tools, and memory, but fewer are making a sharp distinction between:
- shared coordination state
- reasoning behavior
- execution reality

LOOP’s use of Notion is interesting precisely because it does **less** with Notion, more deliberately.
