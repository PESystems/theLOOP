# Contributing

Thanks for taking a look.

This repo is being shared in a real working state, not as a polished framework drop. Some parts are tight. Some parts are still getting shaped in public.

## The most useful feedback

Issues, notes, and critiques are especially helpful when they focus on:
- weak assumptions
- architecture boundary mistakes
- duplication or unnecessary complexity
- places where the system is under-specified
- portability problems
- parts that feel useful in theory but weak in practice

Useful > polite.

## Before suggesting a big rewrite

It helps to be explicit about:
1. what problem you think exists
2. what assumption is causing it
3. what alternative would work better
4. what tradeoff your alternative introduces

## Scope

This repo is currently strongest as:
- a system architecture and operating model
- a structured intake / pipeline concept
- a starting point for human-in-the-loop agent workflows

It is not yet a finished product, framework, or install-and-go tool.

## Community vibe

No shame if something here looks rough or unfinished.
That is part of the point of sharing it early.
